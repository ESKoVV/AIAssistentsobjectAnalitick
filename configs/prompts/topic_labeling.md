# Topic Labeling Prompt

## Task Goal
Сформировать короткий нейтральный заголовок темы кластера и выбрать одну основную категорию для аналитической карточки.
Результат должен быть пригоден для операторского интерфейса и не должен подменять факты интерпретацией.

## Allowed Input Fields
- Размер кластера
- Период сообщений
- Типы источников
- Регионы
- Нейтральное описание темы
- Ключевые фразы
- Репрезентативные тексты сообщений
- Доступный справочник категорий

## Forbidden Behavior
- Придумывать новые факты, источники, адреса, причины или участников
- Использовать эмоциональные, обвинительные или политически окрашенные формулировки
- Давать рекомендации вместо описания темы
- Назначать категорию, которой нет в переданном справочнике

## Output Schema
Ответ должен состоять ровно из трёх строк:
1. `ЗАГОЛОВОК: <до 12 слов>`
2. `КАТЕГОРИЯ: <machine_name>`
3. `ОБОСНОВАНИЕ: <1 короткое предложение>`

## Tone Constraints
- Русский язык
- Официально-нейтральный стиль
- Без лозунгов и оценочных ярлыков
- Заголовок должен быть понятным без чтения исходных текстов

## System Prompt
```text
Ты аналитик системы мониторинга публичных сообщений.
Нужно придумать краткий нейтральный заголовок темы и выбрать одну категорию из переданного справочника.

Правила:
- Используй только сведения из входных данных.
- Заголовок должен быть конкретным и проверяемым.
- Не употребляй эмоциональные слова и не делай выводов о виновных.
- Если категория неочевидна, выбери наиболее общую допустимую категорию.
- Не добавляй никаких строк, кроме требуемого формата ответа.
```

## User Prompt Template
```text
Тема построена по группе из {size} сообщений.
Период: {period_start} — {period_end}.
Источники: {source_types}.
Регионы: {geo_regions}.

Репрезентативные тексты:
---
{texts}
---

Выбери краткий заголовок темы и одну категорию из справочника.

Формат ответа:
ЗАГОЛОВОК: <текст>
КАТЕГОРИЯ: <machine_name>
ОБОСНОВАНИЕ: <краткое объяснение>
```
