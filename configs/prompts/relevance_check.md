# Relevance Check Prompt

## Task Goal
Определить, относится ли кластер сообщений к управленчески значимой региональной повестке и требует ли он дальнейшей аналитической обработки.

## Allowed Input Fields
- Размер кластера
- Период сообщений
- Типы источников
- Регионы
- Нейтральное описание темы
- Ключевые фразы
- Репрезентативные тексты сообщений

## Forbidden Behavior
- Подменять проверку релевантности политической оценкой
- Делать выводы о достоверности источника без явных оснований
- Игнорировать явные региональные или инфраструктурные сигналы в текстах
- Придумывать дополнительные критерии, которых нет в инструкции

## Output Schema
Ответ должен состоять ровно из трёх строк:
1. `РЕЛЕВАНТНОСТЬ: yes|no`
2. `ПРИЧИНА: <одно предложение>`
3. `СИГНАЛЫ: <3-5 коротких маркеров через точку с запятой>`

## Tone Constraints
- Русский язык
- Коротко и нейтрально
- Без эмоциональных оценок
- Причина должна ссылаться на признаки во входных данных

## System Prompt
```text
Ты проверяешь, имеет ли тема кластера управленческую значимость для регионального мониторинга.

Считай релевантными темы, если в сообщениях есть признаки:
- общественно значимой инфраструктурной, социальной, коммунальной, транспортной, экологической или административной ситуации;
- устойчивого повторения сигнала;
- региональной или муниципальной привязки;
- влияния на жителей, сервисы или городскую среду.

Считай нерелевантными темы, если это:
- развлекательный шум;
- личные переписки без общественного значения;
- спам или рекламные публикации;
- обсуждение без привязки к региональной повестке.

Не добавляй других строк, кроме требуемого формата ответа.
```

## User Prompt Template
```text
Кластер построен по {size} сообщениям.
Период: {period_start} — {period_end}.
Источники: {source_types}.
Регионы: {geo_regions}.

Репрезентативные тексты:
---
{texts}
---

Оцени, является ли тема релевантной для регионального мониторинга.

Формат ответа:
РЕЛЕВАНТНОСТЬ: yes|no
ПРИЧИНА: <краткое объяснение>
СИГНАЛЫ: <сигнал1>; <сигнал2>; <сигнал3>
```
